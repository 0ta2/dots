## 目的

ソースを読む**前**に、「著者が何を問い、どう考え、何を結論しているか」の思考構造を
読者が掴める状態にした要約を生成し、Heptabase の指定カードに追記する。

---

## 実行手順

### 1. 入力の確認

ユーザーの指示から以下を把握する:

- **ソース**: URL（WebFetch で取得）またはペーストされたテキスト
- **カード名**: 追記先 Heptabase カードのタイトル（例: "Reading Notes 2026"）
- どちらか不明な場合はユーザーに確認する

### 2. ソースの取得

**URL の場合**: WebFetch でページ全文を取得する。レスポンスが長い場合も要約せず全文を把握してから要約を生成する。

**テキストの場合**: そのまま使う。

出典情報（サイト名・URL・取得日）を控えておく。取得日は `date +%Y-%m-%d` で取得。

### 3. 要約の生成

下記テンプレートを**厳守**する。前置き・後置きの説明文は一切出力しない。
出力言語は**常に日本語**（ソースが英語でも日本語で要約する）。

```markdown
## {{ソースの主題を表す見出し}}

> Source: {{出典名}} | {{URL または "テキスト入力"}} | 取得日: {{YYYY-MM-DD}}

### 中心的な問い
（1–2 文。著者がこのソースで取り組んでいる核心的な問いを抽出する）

### 思考の流れ
（散文 250–450 字。問いに対して、主なアイデア・主張・テーマが互いにどう
積み上がっているかを物語的に説明する。論理の展開・反論への対処・結論への
道筋を含める）

### 要点
- （3–5 項目の箇条書き。最も重要な主張・発見・示唆）

### 著者独自の視点
（1–3 文。このソースを読む価値、他の一般的な解説との差分、著者固有の貢献）
```

**品質基準**:
- 想定読者: その分野を体系的に学ぶ意欲はあるが、まだ専門家ではない人物
- 重要な専門用語は英語原語と日本語訳を併記し、初出時に 1 行で定義する
- 議論を平板にしない。著者の視点の深みを保つ
- 噛み砕きすぎない

### 4. Heptabase カードへの追記

heptabase-cli スキルのコマンドを使い、以下の手順で追記する。

```bash
# 1. カードを名前で検索
heptabase card list -q "{{カード名}}" --limit 10

# 2. jq でカード ID を取得（タイトルが完全一致するものを優先）
# 結果 JSON を確認してカード ID を特定する

# 3. 生成した Markdown を追記
# heptabase card --help でサブコマンドを確認してから実行
heptabase card append --card-id {{cardId}} --content "{{生成した Markdown}}"
```

カードが見つからない場合はユーザーに確認し、必要なら新規カードを作成する:

```bash
heptabase card create --title "{{カード名}}"
```

追記が成功したら、カード名と追記した見出しをユーザーに報告して終了する。
前置き・後置きの余分なコメントは最小限にとどめる。

## エラー対処

| 状況 | 対処 |
|---|---|
| Heptabase デスクトップアプリが起動していない | `heptabase start` を試みるか、ユーザーに起動を依頼する |
| 同名カードが複数ある | タイトル一覧を提示してユーザーに選択を求める |
| URL が取得できない | テキストを貼り付けてもらうようユーザーに依頼する |
| ソース内容が 1 MB を超える | 重要部分を優先し、分割追記を検討する |
